2. **Create cost.py router**:
   - `GET /api/costs/summary` - Main cost summary endpoint:
     - Query system.billing.usage with SKU categorization:
       - `%ALL_PURPOSE%` -> "All-Purpose"
       - `%JOBS%` -> "Jobs Compute"
       - `%SQL%` -> "SQL Warehouse"
       - `%SERVERLESS%` -> "Serverless"
       - else -> "Other"
     - Use established RETRACTION handling: `HAVING SUM(usage_quantity) != 0`
     - Calculate 7-day trend: compare current 7 days vs previous 7 days
     - Aggregate by job_id with SKU breakdown
     - Apply dbu_rate from settings if > 0 to compute dollar amounts

   - `GET /api/costs/by-team` - Team cost rollups:
     - For each job, lookup team tag via Jobs API (batch or cache)
     - Group costs by team
     - Include "Untagged" for jobs without team tag
     - Sort by total_dbus DESC

   - `GET /api/costs/anomalies` - Cost anomalies endpoint:
     - **Cost spikes**: Compare current period DBUs to p90 baseline
       - Flag if current > 2x p90 baseline
       - Return multiplier and reason
     - **Zombie jobs**: Identify using indicators from user decision:
       - High cost with low/zero rows processed (if row metrics available)
       - Long duration with no detectable output
       - Conservative thresholds: >10 DBUs with <100 rows over 30 days
     - Include `job_settings_url` pointing to Databricks job settings

   - Use `asyncio.gather` for parallel query execution where possible
   - Follow billing.py patterns for SQL structure and result parsing
