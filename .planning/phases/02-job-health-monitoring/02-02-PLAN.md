  <action>
Add two endpoints to health_metrics.py router:

1. GET /api/health-metrics/{job_id}/duration
   - SQL query using PERCENTILE_CONT for duration statistics:
     ```sql
     SELECT
       PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY run_duration_seconds) as median_duration,
       PERCENTILE_CONT(0.9) WITHIN GROUP (ORDER BY run_duration_seconds) as p90_duration,
       AVG(run_duration_seconds) as avg_duration,
       MAX(run_duration_seconds) as max_duration,
       COUNT(*) as run_count
     FROM system.lakeflow.job_run_timeline
     WHERE job_id = :job_id
       AND period_start_time >= current_date() - INTERVAL 30 DAYS
       AND run_duration_seconds IS NOT NULL
       AND result_state IS NOT NULL
     ```
   - Set has_sufficient_data = run_count >= 5
   - Return DurationStatsOut

2. GET /api/health-metrics/{job_id}/details
   - Query recent runs (last 10) ordered by start time DESC:
     ```sql
     SELECT run_id, job_id, period_start_time, period_end_time,
            run_duration_seconds, result_state
     FROM system.lakeflow.job_run_timeline
     WHERE job_id = :job_id
       AND period_start_time >= current_date() - INTERVAL 30 DAYS
     ORDER BY period_start_time DESC
     LIMIT 10
     ```
   - Compute is_anomaly for each run: duration_seconds > 2 * baseline_30d_median
   - Get job name from latest_jobs (SCD2 pattern)
   - Get distinct failure reasons from runs with result_state = 'FAILED'
   - Count retries in last 7 days (multiple runs same day)
   - Return JobExpandedOut

Use asyncio.to_thread pattern for SDK calls. Handle NULL durations gracefully.
