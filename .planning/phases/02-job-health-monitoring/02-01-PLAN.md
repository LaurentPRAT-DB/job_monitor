  <action>
Create health_metrics.py router with GET /api/health-metrics endpoint:

1. Accept query param `days: int = 7` (validate 7 or 30 only)

2. SQL query using CTEs (follow pattern from 02-RESEARCH.md):
   - run_stats CTE: Aggregate total_runs, success_count, last_run_time per job_id
   - consecutive_check CTE: Use LAG(result_state) OVER (PARTITION BY job_id ORDER BY period_start_time DESC) to get previous run result
   - priority_flags CTE: Compute priority based on:
     - P1: result_state = 'FAILED' AND prev_result_state = 'FAILED' (2+ consecutive)
     - P2: result_state = 'FAILED' (most recent failed, single)
     - P3: success_rate between 70-89% (yellow zone)
   - retry_counts CTE: Count multiple runs on same day as retries (SUM where run_count > 1)
   - Join all CTEs with latest_jobs (SCD2 pattern from Phase 1)

3. Compute status from success_rate: green >= 90, yellow 70-89, red < 70

4. Sort by priority (P1=0, P2=1, P3=2, None=3) then by success_rate ASC (problem-first)

5. Use asyncio.to_thread pattern for SDK call (established in Phase 1)

Include router in app.py with prefix "/api".
